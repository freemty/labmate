# cc-native-research-template

**让 Agent 实现 scalable 的自我觉察 — Make your research project self-aware through agents.**

> In the age of agents, always ask: can this project achieve one more meta-level of iteration through agents?
>
> 在 agent 的时代，不断思考：任何 project 能不能通过 agent 实现更 meta 一级的迭代？

---

## Why / 为什么

Research projects powered by AI agents share the same infrastructure patterns — experiment folders, prompt versioning, domain paper management, result analysis pipelines, monitoring loops, and the Claude Code harness (skills, agents, hooks) that ties it all together.

每个 AI agent 驱动的研究项目都在重复造同一套轮子：实验目录、prompt 版本管理、论文阅读、结果分析、监控循环、以及把这些串起来的 Claude Code 基础设施。

**We built three high-intensity research projects** (fars-reviewer, PostTrainBench, hle-solver) and found ourselves reinventing this infrastructure every time. Specs rewritten, agents re-designed, hooks re-configured.

**我们做了三个高强度研究项目**，每次都在重新搭脚手架。是时候把共性抽象成模板了。

## What / 是什么

A **GitHub template repo** that gives you a complete Claude Code research harness out of the box:

一个开箱即用的 Claude Code 研究项目 harness 模板：

- **7 Agents** with detailed system prompts and least-privilege boundaries
- **7 Skills** for experiment lifecycle, analysis, knowledge management, and presentation
- **5 Hooks** enforcing a "remind, don't block" workflow discipline
- **Experiment infrastructure** with standardized naming, scripts, and shared utilities
- **Auto-maintained project knowledge** that survives context compaction
- **Slides & visualization** generated by background agents

The template produces **infrastructure, not content**. Your research is yours.

模板产出的是**基础设施，不是内容**。你的研究是你的。

| Agent | Model | Role |

| **project-advisor** | opus | Experiment history, research findings, codebase navigation |
| **cc-advisor** | sonnet | Claude Code workflow best practices (9-source knowledge base) |

| **slides-maker** | sonnet | Generate HTML presentations (runs in background) |
| **exp-manager** | sonnet | Monitor experiments, diagnose failures, retry jobs |

## Skills

| Skill | Trigger | What it does |
|-------|---------|-------------|
| `/new-experiment` | Starting a new experiment | Scaffold `exp/{id}/` with README, config, scripts |
| `/analyze-experiment` | Experiment completed | Run analysis → @domain-expert → @slides-maker |
| `/update-project-skill` | Knowledge stale (>24h) | Opus scans project → regenerates SKILL.md |
| `/present-template` | Need overview slides | @template-presenter → @slides-maker |
| `/weekly-progress` | Friday reminder | CHANGELOG + git log → `docs/weekly/` |
| `/commit-changelog` | Ready to commit | Standardized commits + CHANGELOG |

## Quick Start / 快速开始

### 1. Use this template

Click **"Use this template"** on GitHub → create your repo.

### 2. Bootstrap

```bash
cd your-new-repo
bash bootstrap.sh
```

Four questions: project name, description, research domain, compute environment.

四个问题：项目名、描述、研究领域、计算环境。

### 3. First experiment

```
/new-experiment
```

Creates `exp/exp01a/` with README, config, run.py, analyze.py.

### 4. Run the loop

```
python exp/exp01a/run.py --config exp/exp01a/config.yaml
/analyze-experiment
/update-project-skill
```

**Full cycle:** `dev` → `/new-experiment` → run → `/analyze-experiment` → commit → `/update-project-skill` → repeat.

## Architecture / 架构

```
your-project/
├── CLAUDE.md                     # Route hub (<80 lines)
├── .claude/
│   ├── agents/                   # 7 agent definitions
│   ├── skills/                   # 7 skills
│   ├── hooks/                    # 5 hook scripts
│   └── settings.local.json      # Permissions + hook config
├── exp/                          # Experiments (exp{NN}{x} naming)
│   ├── lib/                      # Shared analysis utilities
│   └── summary.md                # Cross-experiment flight recorder
├── docs/papers/                  # Domain papers + landscape.md
├── prompts/                      # Versioned prompt templates (_v{NN}.md)
├── scripts/                      # Launch, monitor, download
├── slides/                       # Generated presentations
│   └── references/               # Visual spec (frontend-slides + agent-slides)
├── viewer/                       # Flask analysis dashboard
├── bootstrap.sh                  # First-run personalization
└── .pipeline-state.json          # Workflow state machine
```

### Design Principles / 设计原则

- **Least privilege** — 4/7 agents are read-only, 2 have scoped write directories
- **Remind, don't block** — Hooks suggest the right action, never hard-block
- **Context protection** — Heavy output (slides HTML) generated in background agents
- **Repo as record system** — All knowledge version-controlled, nothing in chat history only
- **Agent-readable first** — Optimize for agent comprehension, not human aesthetics

## Research Principles / 搜索原则

1. **Measure first** — Attack the measured bottleneck, not intuition
2. **Baseline is sacred** — Every claim needs a reproducible baseline comparison
3. **Statistical significance** — Single-run results are anecdotal
4. **Ablation-driven** — Multi-factor changes require per-factor isolation
5. **Respect negative results** — Don't retry ❌ directions without new evidence
6. **Predict first** — Record expected values before running, calibrate after

## Applying to Existing Repos / 迁移已有项目

Already have a research repo? You don't need to start over.

```bash
# 1. Copy the .claude/ infrastructure
cp -r template/.claude/ your-repo/.claude/

# 2. Copy supporting files
cp template/bootstrap.sh your-repo/
cp -r template/slides/references/ your-repo/slides/references/

# 3. Bootstrap
cd your-repo && bash bootstrap.sh

# 4. Initialize project knowledge (deep scan of existing repo)
/update-project-skill
```

The bootstrap mode of `/update-project-skill` will scan your existing codebase, experiments, and docs to generate the initial project knowledge.

## The Vision / 愿景

The first user of this template is [PostTrainBench](https://github.com/freemty/PostTrainBench) — an agent that iterates on auto post-training. The agent runs experiments, analyzes results, reads papers, updates its own knowledge, and decides what to try next.

这个模板的第一个用户是 PostTrainBench — 一个迭代 auto post-training 的 agent。它跑实验、分析结果、读论文、更新自己的知识、决定下一步做什么。

**This is what scalable self-awareness looks like:** the agent doesn't just execute tasks — it maintains the infrastructure that makes it smarter over time.

**这就是 scalable 的自我觉察：** agent 不只是执行任务，它维护着让自己随时间变得更聪明的基础设施。
